## **Overview**

RPAS-CM v2.3.0 is a milestone release that introduces **Semantic Governance**. By operationalizing the DRACO Board locally, the framework can now detect and block "Mythos-Class" failure modes (unauthorized initiative, semantic drift, and sandbox breakouts) during the orchestration phase.

---

## **Key Changes**

### **1. Gate 5: Semantic Integrity (DRACO)**
- **Feature**: Localized multi-agent Review Board (Evidence, Governance, Challenger).
- **Automation**: Triggered via `pnpm run draco:preflight`.
- **Impact**: Provides an independent audit of the `git diff` against the stated intent.

### **2. G5-BLOCKING-MODE Rule**
- **Architecture**: Ritual-aware enforcement.
- **Phases 1-2**: Advisory mode (Safety feedback only).
- **Phase 3**: Mandatory Blocking mode (Halt execution on risk).

### **3. Mythos-Class Prevention**
- **Hardening**: Specifically tuned to detect "Shadow Initiative" (e.g., external network artifacts, unasked-for data exports) before they are committed.

---

### **4. UX Transparency & Cognitive Layer**
- **Feature**: Structured "DRACO Board Reports" showing Evidence, Governance, and Challenger deliberations.
- **Traceability**: Active latency notices in `PRE-001` and `validate-rpas.ps1` to manage the ~90s semantic safety cycle.
- **FAQ**: Hardened `README.md` with active guidance on Phase 3 duration and purpose.
