## **Overview**

RPAS-CM v2.2.0 represents a significant advancement in governance maturity, transitioning the framework from reactive manual attestation to **automated machine-verifiable enforcement**. This release specifically addresses the emergent risks of autonomous AI behaviors (e.g., Project Glasswing) by formalizing the system's structural immunity.

---

## **Key Changes**

### **1. Automated Gate 4 (Governance Attestation)**
- **Feature**: Machine-verifiable schema enforcement via AJV.
- **Impact**: Attestations are now structurally and semantically audited against the `TAR-COL` protocol before a change can be certified.
- **Constraint**: Mandatory `rpas-attestation.json` for all non-hygiene tasks.

### **2. Structural Baseline Consolidation**
- **Action**: Relocated 11 authoritative governance artifacts to the dedicated `/governance` directory.
- **Benefit**: Improved audatibility, clean tier separation, and deterministic rule discovery for agents and human governors.

### **3. Glasswing Immunity Addendum**
- **Addition**: Formalized documentation of the five defensive layers (G1–G5) preventing autonomous mutation/executuion.
- **Context**: Explicitly neutralizes failure modes associated with Claude Mythos unverified patches and sandbox escapes.

## **Upgrade Path**

1. Ensure `ajv` and `ajv-formats` are installed in the workspace root.
2. All new proposals must include a `governance/rpas-attestation.json` file.
3. Reference `RPAS.md v2.2.0` for updated authority boundary definitions.
