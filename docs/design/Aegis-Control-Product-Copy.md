## Core Principles

- **Always Verified** — State is validated using independent evidence, not inferred from telemetry alone.
- **Deterministically Enforced** — Policies are executable and testable, not descriptive or advisory.
- **Continuously Restored** — Deviation triggers guided remediation until the intended state is proven.

---

## Product Definition (Aegis Control)

Aegis Control is a Digital Governance Control System built on the ICT Governance Framework, powered by the ADPA Governance Engine.

It introduces a closed-loop system that continuously:

- Defines intended state
- Verifies actual state using independent evidence
- Enforces policy deterministically
- Orchestrates controlled remediation
- Produces proof of closure

---

## Operational Loop

1. **Define Intended State** — Requirements formalized into structured governance artifacts.
2. **Verify with Independent Evidence** — Multi-source evidence chains with confidence scoring.
3. **Enforce and Remediate (L4 controlled)** — Approval-gated orchestration; human-in-the-loop execution.
4. **Close with Proof** — Remediation complete only when evidence meets thresholds and lineage is recorded.

---

## Proof Clause (Canonical)

> Remediation is not complete until independent evidence confirms consistent, high-confidence state — with full lineage from requirement to runtime.
