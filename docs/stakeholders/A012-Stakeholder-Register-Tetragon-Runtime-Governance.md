## Purpose
This stakeholder register documents the key stakeholders for the Tetragon runtime governance pilot and integration into the ICT Governance Framework.

## Stakeholders

| Stakeholder | Role | Interest | Influence | Engagement Approach |
|-------------|------|----------|-----------|---------------------|
| CIO / Executive Sponsor | Sponsor | Endorse strategic runtime governance and approve pilot funding | High | Executive briefings, approval checkpoints |
| ICT Governance Program Lead | Project Sponsor | Ensure integration aligns with governance strategy | High | Weekly status reviews, steering committee updates |
| Security Operations Lead | Primary user | Use runtime events for incident detection and investigation | High | Collaborative design sessions, pilot demos |
| Platform Engineering Lead | Implementation lead | Ensure runtime deployment is safe and reliable | High | Technical design reviews, environment readiness validation |
| DevOps/Kubernetes Team | Deployment owner | Deploy and support Tetragon on target cluster | Medium | Deployment coordination, runbook handover |
| Compliance/Risk Manager | Audit reviewer | Validate runtime evidence supports policy and compliance requirements | Medium | Compliance review sessions, evidence mapping workshops |
| Application Owner | Data owner | Ensure runtime governance does not disrupt application workloads | Medium | Change communication, operational coordination |
| Infrastructure Operations | Operations support | Monitor cluster health and manage host platform impact | Medium | Incident readiness briefings, support handover |
| Security Architecture | Architecture oversight | Confirm runtime integration matches security architecture principles | Medium | Architecture review, design approval |
| Governance Analysts | Reporting and measurement | Map runtime events to governance dashboards and metrics | Medium | Workshop sessions, dashboard definition reviews |

## Stakeholder Roles and Responsibilities

- CIO / Executive Sponsor: Approve pilot scope and secure organizational support.
- ICT Governance Program Lead: Guide the integration and ensure alignment with governance objectives.
- Security Operations Lead: Define detection use cases and validate event relevance.
- Platform Engineering Lead: Assess environment readiness, approve kernel/eBPF support, and oversee deployment.
- DevOps/Kubernetes Team: Implement pilot deployment and manage runtime workloads.
- Compliance/Risk Manager: Confirm pilot output meets compliance evidence needs.
- Application Owner: Coordinate application context and minimize operational disruption.
- Infrastructure Operations: Support cluster health monitoring and remediation.
- Security Architecture: Validate design, enforcement model, and risk posture.
- Governance Analysts: Define dashboard mapping, metrics, and stakeholder reporting.

## Engagement Plan

- Initial briefing: share the ideation concept and business case with executive, security, and platform stakeholders.
- Technical review: conduct a Tetragon readiness and compatibility review with the platform and security teams.
- Pilot planning: align on scope, target cluster, detection categories, and data ingestion model.
- Evaluation checkpoint: review pilot results, incident correlation value, and stakeholder feedback.
- Decision briefing: present a recommendation for continued integration or phased rollout.

## Notes

- This register will be maintained throughout the pilot and updated when new stakeholders or governance owners are identified.
- Stakeholder influence and interest should be reviewed before each major pilot decision.

*This stakeholder register supports the successful planning and execution of the Tetragon runtime governance pilot by clearly defining stakeholder roles, interests, and engagement activities.*
